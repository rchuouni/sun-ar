# 太陽軌跡AR — UI/UX刷新仕様書（v7）

実装担当モデルへ：本書は動作中の v6（sun-ar-v6.html）を土台に、UI/UXを引き上げるための仕様である。**ロジックの再設計は不要**。「壊してはいけないもの」を最初に示すので、必ず守ること。

---

## 0. 壊してはいけないもの（最重要・実機検証済みの資産）

1. **センサー許可フロー**：`DeviceOrientationEvent.requestPermission()` は「センサーを有効化」ボタンの**タップハンドラ先頭で同期的に**呼ぶこと。前に await / setTimeout / 他の許可要求を挟むと iOS が user gesture と認めず、許可ダイアログが出なくなる（実機で確認済みの不具合）。2段階フロー（STEP1センサー → STEP2カメラ）の構造を維持する。
2. **手動北合わせのロジック**：センサーの絶対方位（webkitCompassHeading）は北合わせに**使わない**。「北に設定」押下時のカメラ水平方位を `atan2(fwd.x, -fwd.z)` で取り、`northOffset = -headingNow` とする方式を維持（磁気偏角バイアス約9°を相殺する解として実機確認済み）。
3. **座標系**：Y上、北=-Z、東=+X。SunCalcのazimuthは南基準なので `+π` して北基準に変換している。変更しない。
4. **Safari固有の対策**：`playsinline` 属性、`preserveDrawingBuffer:true`、`renderer.setSize(w,h,false)`、pixelRatio上限2、`-webkit-backdrop-filter` 併記、`env(safe-area-inset-*)`、`100svh` 相当の扱い。
5. **localStorage** による northOffset の保存・復元。
6. 単一HTMLファイル構成（CDN: three.js r128 / suncalc 1.9.0）。ビルド不要を維持。

## 1. デザイン原則

- **空が主役**：UIは「計器」。AR表示（カメラ映像＋太陽）を遮らない。常設UIの画面占有率は合計15%以下。
- **片手・屋外**：このアプリは屋外で空にかざして使う。すべての操作ターゲットは44pt以上、主要操作は画面下1/3に集約。直射日光下でも読めるコントラスト（白文字は背景スクリム必須）。
- **専門値より意味**：「α β γ」のような生センサー値はデフォルト非表示。ユーザーに見せるのは「日の出 5:32 / 南中 11:48 / 日の入 18:04」のような意味のある情報。
- **モードレス**：較正・日付変更などの操作中もAR表示は生き続ける。全画面モーダルは北合わせガイドのみ。

## 2. 画面構成（情報アーキテクチャ）

```
[オンボーディング]  STEP1 センサー → STEP2 カメラ → (拒否時リカバリ)
[メイン/AR]
  ├ 上端: ステータスピル（中央・小型・タップで詳細）
  ├ 右上: カメラ切替 🔄（カメラ使用時のみ）
  ├ 中央: 太陽ファインダー（太陽が画面外のとき方向矢印）
  ├ 下部: サンインフォバー（日の出/南中/日の入 ＋ 現在の太陽az/alt）
  └ 最下: ツールバー 4ボタン（☀️軌道 / 📅日時 / 🧭較正 / ⚙️設定）
[シート] 画面下から最大55vh。1枚のみ開く。ハンドルバー付き、下スワイプでも閉じる
[北合わせガイド] 全画面オーバーレイ（既存フロー踏襲＋改良）
```

### v6からの主な変更点一覧

| v6 | v7 |
|---|---|
| 上端の常時デバッグステータスバー | 廃止。小型ステータスピルに置換、ログは設定内に移動 |
| HUD左上に生センサー値パネル常設 | 廃止。ユーザー向けサンインフォバーに置換。生値は設定内「診断」でON可 |
| 📊ボタン（情報トグル） | ⚙️設定に変更（診断表示・北設定リセット・チュートリアル再表示を格納） |

| 時刻スライダーのみ | スライダー＋再生ボタン（時間を自動で流すアニメーション）|

## 3. コンポーネント仕様

### 3.1 オンボーディング（構造維持・表現改良）

- 進捗ドット（● ○）をSTEP表記に併設。
- STEP1の説明文に「なぜ必要か」を1行で：「現実の空と方位を一致させるために使います」。
- 許可ダイアログ待ちの間、ボタンをスピナー化（連打防止 `disabled`）。
- STEP2にプレビュー的なイラスト不要。既存の絵文字アイコンで可。
- 成功時の「センサー受信OK ✓」は維持（ユーザーの安心材料として機能していた）。

### 3.2 ステータスピル（新設）

- 位置：上端中央、`top: max(10px, safe-area)`。高さ28px、左右paddingで内容に追従。
- 通常時：`● 追跡中`（cyanドット）。異常時：`● センサー未受信`（redドット）。較正未実施時：`● 北が未較正`（amberドット、タップで🧭シートを開く）。
- タップで詳細ポップ（位置・高度・コンパス参考値・fps）を3秒表示。

### 3.3 サンインフォバー（新設・HUDの代替）

- 位置：ツールバーの直上。横幅 `calc(100% - 24px)`、高さ約56px、パネル様式。
- 内容（左→右）：
  - `日の出 05:32` / `南中 11:48` / `日の入 18:04`（SunCalc.getTimes で算出）
  - 右端に現在の太陽 `az 247° / alt 31°`（tabular-nums）
- 太陽が地平線下のときは右端を `🌙 地平線下` 表示に切替。
- タップでバー自体を最小化（右端の小さなタブだけ残す）⇄ 復元。状態はlocalStorageに保存。

### 3.4 太陽ファインダー（新設・UX目玉）

- 太陽（現在 or 指定時刻のうち表示中のもの）がカメラ視野外のとき、画面端に**方向矢印＋太陽アイコン**を表示し、どちらへ端末を向ければよいか案内する。
- 実装：太陽のワールド座標をカメラ空間へ射影。NDCが |x|>1 または |y|>1 または z>1（背後）なら視野外。画面中心から太陽方向へのベクトルを正規化し、画面端（マージン24px）にクランプした位置に矢印を置き、`atan2` で回転。
- 矢印は amber（現在の太陽）/ purple(指定時刻)。視野内に入ったらフェードアウト（0.3s）。
- 毎フレーム実行されるためDOM生成はせず、固定要素のtransform更新のみで実装すること。

### 3.5 シート共通

- 上部に38×5pxのハンドルバー。下方向スワイプ（touchmove 60px超）で閉じる。
- 開閉アニメーションは既存の `cubic-bezier(.22,1,.36,1) 0.28s` を維持。
- シートを開いてもARとサンインフォバーは更新を継続。
- 背景タップでは閉じない（屋外での誤操作防止。閉じる操作はハンドル・スワイプ・同ボタン再タップの3系統）。

### 3.6 ☀️ 季節の軌道シート（小改良）

- 既存のチェックボックス4行を維持。各行の右端に**その日の南中高度**を追記（例: `夏至 6/21 · 南中78°`）。一目で弧の高さの違いが予想できる。
- 先頭に「今日」の行を追加（常時ON・無効化チェックボックス・amber）。今日の軌道が常に出ていることを明示する。

### 3.7 📅 日時シート(操作性強化)

- プリセットチップ・日付ステッパー・時刻ステッパーは現行踏襲。ボタン高さを42→48pxに。
- **再生ボタン（新設）**：時刻スライダー左に ▶ を配置。押すと customMin を毎フレーム+2分進め（約1分/秒）、紫の太陽が軌道上を滑らかに移動。再タップ／スライダー操作で停止。24:00到達で0:00へループ。
- スライダーには 0/6/12/18/24 の目盛りラベルを下に併記。
- 指定時刻の太陽（紫）には小さな時刻ラベルスプライト（例 `14:30`）を追従させる。ラベルはCanvasTextureで生成し、時刻変更時のみ再描画（毎フレーム再生成禁止）。

### 3.8 🧭 較正シート＋北合わせガイド（文言・誘導改良）

- シート内は「🎯 手動で北を上にする」（primary）と「📱 コンパスアプリ手順」（折りたたみ、現行内容）と微調整スライダーのみ。v6で削除した自動北合わせは復活させない。
- ガイドオーバーレイの改良:
  - 手順を画面上部に1行ずつ表示：「1. コンパスアプリ等で実際の北を確認 → 2. iPhoneをその方向へ → 3. 下のボタン」。
  - 中央の赤線に加え、**水平インジケータ**を追加：端末が水平から±15°以上傾いていると「端末を立てて水平に」と注意表示し、設定ボタンを半透明に（誤較正防止。判定は `beta` が 75–105° 帯にあるか等、実装時に調整可）。
  - 設定成功時：画面全体に短いフラッシュ（amber 10% 0.4s）＋toast「北を設定しました ✓」＋ステータスピルが `● 追跡中` に変化。
- 微調整スライダーはハプティック代替として5°刻みのスナップ感（value表示の強調）を入れる。

### 3.9 ⚙️ 設定シート（新設）

- 「診断表示」トグル：ONで旧HUD（センサー生値・fps・ログ）を表示。デフォルトOFF。
- 「北設定をリセット」：localStorage消去＋northOffset=0。確認トースト付き。
- 「チュートリアルをもう一度」：オンボーディングSTEP1を再表示はせず、簡単な使い方カード（3項目）をシート内に展開する程度でよい。
- バージョン表記 `v7`。

### 3.10 カメラ切替 🔄

- 現行の右上配置を維持。サイズ48px。切替中はアイコンをスピナー化。
- 内カメラ時はサンインフォバー左端に `前面カメラ — AR方位は背面基準` の小さな注意テキストを出す（既知の制約の明示）。

## 4. マイクロインタラクション

- ボタン類 `:active` で `transform: scale(.96)` + 背景明度アップ（0.1s）。
- toast は既存様式を継承。同時に1つまで、新しいものが置き換える。
- 太陽（現在）の球に呼吸的な微パルス（glow opacity 0.16⇄0.24、周期4s）。派手にしない。
- prefers-reduced-motion 対応：パルス・フラッシュ・再生アニメ以外の装飾的動きを無効化。

---

## 5. アクセシビリティ / 堅牢性

- 全ボタンに `aria-label`。シートは `role="dialog"` `aria-modal="false"`。
- 文字サイズ最小11px、重要数値は14px以上。
- 例外時フォールバックは現行どおり：カメラ失敗→3Dのみ、位置失敗→東京、SunCalc失敗→ログ表示。エラーはログ（設定>診断）に集約し、ユーザー向けにはtoastで平易に。
- 画面回転（orientationchange）時のレイアウト崩れがないこと。シートは横向きでは `max-height:80vh`。

---

## 6. 受け入れ基準（実装後チェックリスト)

1. STEP1で許可ダイアログが従来どおり表示される（最重要・回帰確認）
2. 手動北合わせ後、西を向いてWラベルが実方位と±3°以内（回帰確認）
3. 太陽が視野外のとき矢印が正しい方向を指し、視野内でフェードアウトする
4. サンインフォバーの日の出/南中/日の入が現在地で正しい
5. ▶再生で紫太陽が軌道上を滑らかに周回し、停止・ループが機能する
6. 診断表示OFFの初期状態で、画面上の常設UIがピル＋インフォバー＋ツールバー＋🔄のみである
7. シートが下スワイプで閉じる
8. 北設定リセット→再較正の一連が機能する
9. iPhone Safari 実機で30fps以上を維持（毎フレームのDOM生成・テクスチャ生成がないこと)

## 7. 実装メモ

- v6のJS構造（即時関数＋state＋bindControls）をそのまま拡張してよい。フレームワーク導入禁止（単一HTML維持）。
- SunCalc.getTimes(date, lat, lon) で sunrise / solarNoon / sunset を取得。位置取得後に1回計算し、日付が変わったときのみ再計算。
- 太陽ファインダーの射影は `vector.project(camera)` を使用。world.rotation.y（northOffset）が掛かったワールド座標で行うこと（生のsunVec値に手動でoffsetを足さない。worldグループ配下の実位置を `getWorldPosition` で取るのが安全）。
- ファイル名は index.html 互換（Netlify上書きデプロイ運用）。
